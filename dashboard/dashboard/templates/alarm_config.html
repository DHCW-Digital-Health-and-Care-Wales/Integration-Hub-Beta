How Alarm 1 works
The alarm fires when an HL7 server has received no messages for longer than the period threshold for the current time window (Day / Evening / Weekend). Once fired, it will not re-fire until the Alerting Gap has elapsed — this prevents repeated alerts for the same ongoing outage.
