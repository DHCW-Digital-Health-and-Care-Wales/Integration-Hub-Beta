How Alarm 1 works
The alarm fires when an HL7 server has received no messages for longer than the period threshold for the current time window (Day / Evening / Weekend). Once fired, it will not re-fire until the Alerting Gap has elapsed — this prevents repeated alerts for the same ongoing outage.

How it works
Alarm 1 queries Application Insights for the most recent MESSAGE_RECEIVED event from each enabled HL7 server. If the time since the last message exceeds the configured threshold, the server is flagged as In Alarm.

Servers with no data in the last 30 days are shown as Unknown. The threshold can be set independently per server — for example, a server that only processes batch overnight jobs may need a longer threshold than one handling real-time ADT feeds.
